[23.05.2026 23:12] justi: Проблема была в том что OnlySq в streaming режиме возвращает tool_calls с пустыми аргументами:

{"tool_calls": [{"function": {"name": "bash", "arguments": "{}"}}]}
А реальные аргументы ({"command": "ls /root"}) приходят только в non-streaming ответе.

Что мы сделали:

1. В openai_compat.py добавили словарь _fallback_tool_args = {}

2. После создания streaming соединения делаем параллельный non-streaming запрос и сохраняем аргументы:
_nb_resp = await self._client.chat.completions.create(..., stream=False)
_fallback_tool_args[tc.function.name] = tc.function.arguments
3. При обработке streaming tool_call — если arguments пустые, берём из fallback:
if not _args or _args.strip() in ("{}", ""):
    _args = _fallback_tool_args.get(tc.function.name, "{}")
По сути каждый запрос теперь делается дважды — streaming для Claude Code и non-streaming для получения реальных аргументов. Поэтому RPM расходуется в 2 раза быстрее.
